# KeyChecker

Self-hosted веб-сервис для команды: вставляешь пачку API-ключей LLM-провайдеров
и получаешь их валидность, а где API это отдаёт — баланс. Проверка разовая и
stateless: ключи нигде не сохраняются, живут только в памяти на время запроса.

## Возможности

- Ввод ключей пачкой (по одному на строку); провайдер определяется
  автоматически по префиксу.
- Параллельная проверка с ограничением конкурентности; результаты — в порядке
  ввода.
- Защита общим Bearer-токеном (fail-closed: без токена сервис не стартует).
- Один Docker-образ: Go-бэкенд отдаёт встроенный React/Vite фронт и `/api`.

## Провайдеры

| Провайдер   | Префикс ключа       | Валидность           | Баланс по API            |
| ----------- | ------------------- | -------------------- | ------------------------ |
| OpenAI      | `sk-` / `sk-proj-`  | GET `/v1/models`     | Недоступен               |
| Anthropic   | `sk-ant-`           | GET `/v1/models`     | Недоступен               |
| OpenRouter  | `sk-or-`            | GET `/api/v1/key`    | **Да** (остаток/лимит)   |
| Perplexity  | `pplx-`             | минимальный chat     | Недоступен               |
| FAL.ai      | `id:secret`         | авторизованный запрос| Недоступен               |

Баланс по API отдаёт только OpenRouter; для остальных показывается статус
валидности и пометка «баланс недоступен по API».

> Точные эндпоинты валидации Perplexity и FAL.ai взяты по документации на
> момент разработки и опираются на статус-код ответа; при изменении их API
> достаточно поправить тело `Check` в соответствующем файле провайдера.

## Запуск через Docker

```bash
docker build -t keychecker .
docker run -d -p 8080:8080 -e KEYCHECKER_TOKEN=<секретный-токен> keychecker
```

Открой `http://localhost:8080`, введи токен и ключи.

## Локальная разработка

Бэкенд:

```bash
export KEYCHECKER_TOKEN=devtoken
go run ./cmd/keychecker
```

Фронт (dev-сервер с проксированием `/api` на бэкенд):

```bash
cd web
npm install
npm run dev
```

## Безопасность

- Ключи не логируются, не пишутся на диск и не кэшируются; в выводе всегда
  маскируются.
- Исходящие запросы идут только на захардкоженные хосты провайдеров.
- Сервис рассчитан на доверенную команду за TLS-прокси/VPN, не на публичный
  доступ.
