# DEPLOYMENT

Deployment status: **N/A**

KeyChecker поставляется как Docker-образ, который команда разворачивает
на собственной инфраструктуре. Управляемого продакшн-деплоя из этого
репозитория нет.

## Как развернуть

```bash
docker build -t keychecker .
docker run -d -p 8080:8080 -e KEYCHECKER_TOKEN=<секретный-токен> keychecker
```

Обязательные переменные окружения:

- `KEYCHECKER_TOKEN` — общий Bearer-токен доступа. Без него сервис не
  стартует (fail-closed). Не коммить и не логировать это значение.
- `KEYCHECKER_ADDR` — адрес прослушивания (по умолчанию `:8080`),
  опционально.

## Предпосылки перед деплоем

- Сервис доступен только доверенной команде (за VPN/обратным прокси с
  TLS); он не рассчитан на публичный интернет.
- Токен сгенерирован криптостойко и передан участникам вне репозитория.

Любой запрос на «деплой» требует, чтобы этот runbook был `READY` для
целевой среды; сейчас он `N/A` — управляемого деплоя нет.
